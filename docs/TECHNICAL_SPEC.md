## 二、核心功能流程

### 2.1 录音转写流程

```
用户点击"开始录音"
    ↓
Swift服务启动AVAudioEngine
    ↓
音频数据实时转写（ whisper.cpp ）
    ↓
用户点击"停止录音"
    ↓
保存完整音频文件
    ↓
发送audio_saved消息
    ↓
前端显示转写结果
    ↓
后台候选股票匹配
    ↓
保存笔记到 data/stocks/{code}.md
```

### 2.2 AI 纠错环节（v3.1 新设计）

**设计理念**：AI只做辅助，不做分析和提炼

**职责范围**：
- 纠正错别字和同音字错误
- 纠正常见的股票名称错误
- 不提炼、不分析、不添加内容

**Prompt 示例**：
```
请纠正以下录音转写文本中的错误：
{text}

候选股票：
{candidates}

任务：
- 纠正错别字和同音字错误
- 根据候选股票列表，纠正常见的股票名称错误
- 不要改变原意，只做文字纠正

只返回纠正后的文本。
```

### 2.3 股票名称匹配

**匹配流程**：
```
转写文本: "中原海浪今天尾盘拉升"
    ↓ 提取中文片段
片段: ["中原海浪", "今天", "尾盘", "拉升"]
    ↓ 编辑距离匹配
匹配: "中原海浪" → "中原海能" (distance=1, confidence=80%)
    ↓ 生成候选列表
候选: [{ segment: "中原海浪", stock: { code: "000544", name: "中原海能" }, confidence: 0.8 }]
    ↓ 返回给前端
显示: 识别股票: 中原海能 (000544) 80%
```

## 六、已知问题

### 6.1 录音异常问题

**现象**：录音刚开始就结束，无法正常录制

**可能原因**：
- Swift服务与Electron的WebSocket通信问题
- whisper.cpp加载时间过长导致超时

**排查建议**：
1. 检查Swift服务日志
2. 重启语音服务
3. 检查麦克风权限

### 6.2 Whisper转写质量

**问题**：专有名词（股票名称）识别不准确

**解决方案**：
- 使用编辑距离算法进行后处理匹配
- AI环节辅助纠错
