## 一、产品概述

### 1.1 产品定位

股票投资语音笔记系统是一款专为股票投资者设计的智能笔记管理工具，通过语音录入和AI处理，帮助投资者高效记录投资想法、操作理由及后市观点，为后续复盘分析和成功率验证奠定基础。

### 1.2 目标用户

- **主要用户**：个人股票投资者（中短线交易者为主）
- **用户特征**：
  - 每日看盘时间 > 2小时
  - 有记录投资想法的习惯或需求
  - 希望通过复盘提升投资水平
  - 对AI辅助工具有接受度

### 1.3 核心价值

| 价值点 | 描述 |
|--------|------|
| **效率提升** | 语音录入解放双手，不影响看盘操作 |
| **结构化记录** | 以股票为对象、时间为维度组织笔记 |
| **智能处理** | AI自动优化文本、提取观点 |
| **可追溯性** | 完整时间轴记录，支持复盘分析 |

### 2.2 功能详细说明

#### 2.2.1 笔记管理

**功能描述：** 以股票为核心对象组织笔记，一个股票对应一个笔记文件，文件内按时间轴记录每次录入。

**功能列表：**

| 功能 | 优先级 | 描述 |
|------|--------|------|
| 创建笔记 | P0 | 输入股票代码，创建该股票的笔记文件 |
| 编辑笔记 | P0 | 编辑笔记内容，支持Markdown格式 |
| 保存笔记 | P0 | 自动保存到本地文件 |
| 删除笔记 | P1 | 删除指定时间节点的记录 |
| 搜索笔记 | P1 | 按股票代码、关键词搜索 |
| 导出笔记 | P2 | 导出为Markdown/PDF格式 |

**数据格式：**

```markdown
---
stock_code: "600519"
stock_name: "贵州茅台"
created_at: "2024-01-15T09:30:00+08:00"
updated_at: "2024-01-16T15:20:00+08:00"
total_entries: 12
---

# 贵州茅台投资笔记

---

## 📅 2024-01-16

### 🕐 15:20 午盘观察

> **观点**: 看多 (信心: 0.8) | **周期**: 短线

午后继续走强，成交量持续放大...

*音频: [20240116-1520.wav](../audio/600519/20240116-1520.wav) (45秒)*
```

#### 2.2.2 时间轴

**功能描述：** 以可视化时间轴展示所有笔记记录，支持按时间、股票、观点筛选。

**功能列表：**

| 功能 | 优先级 | 描述 |
|------|--------|------|
| 时间线展示 | P0 | 可视化展示所有笔记时间节点 |
| 按股票筛选 | P0 | 只显示指定股票的记录 |
| 按观点筛选 | P1 | 筛选看多/看空/中性观点 |
| 时间范围选择 | P1 | 选择日期范围查看 |
| 点击跳转 | P0 | 点击时间节点跳转到对应笔记 |
| 缩放拖拽 | P1 | 支持时间轴缩放和拖拽 |

**交互设计：**

- 时间轴按日期分组，每日内按时间排序
- 不同观点用颜色区分：看多（红色）、看空（绿色）、中性（灰色）
- 支持鼠标滚轮缩放时间范围
- 双击时间节点打开详细编辑

#### 2.2.3 语音录入

**功能描述：** 支持实时录音和音频文件导入，自动转写为文字。

**功能列表：**

| 功能 | 优先级 | 描述 |
|------|--------|------|
| 实时录音 | P0 | 点击录音按钮开始录音 |
| 录音可视化 | P1 | 实时显示音量波形 |
| 停止录音 | P0 | 点击停止，保存音频文件 |
| 音频导入 | P1 | 导入外部音频文件 |
| 语音转文字 | P0 | 自动将音频转写为文字 |
| 音频回放 | P2 | 播放已录制的音频 |

**技术要求：**

- 音频格式：WAV（16kHz，单声道）
- 支持格式转换：MP3、M4A、WebM转WAV
- 中文识别准确率 > 95%
- 转写延迟 < 3秒/分钟音频

#### 2.2.4 AI处理

**功能描述：** 集成本地和云端AI能力，提供文本优化、观点提取等功能。

**功能列表：**

| 功能 | 优先级 | 描述 |
|------|--------|------|
| 文本优化 | P0 | 去除口语化表达，修正语法 |
| 观点提取 | P0 | 自动提取投资观点和信心度 |
| 关键词提取 | P1 | 自动提取关键词标签 |
| 摘要生成 | P1 | 生成笔记摘要 |
| 模式切换 | P0 | 本地/云端/智能模式切换 |

**AI模式说明：**

| 模式 | 描述 | 适用场景 |
|------|------|---------|
| 本地模式 | 使用本地Ollama运行，零成本 | 隐私敏感、无网络 |
| 云端模式 | 调用云端API，能力更强 | 复杂任务、高质量要求 |
| 智能模式 | 自动选择最优服务 | 日常使用 |

**支持的云端服务商：**

| 服务商 | 模型 | 价格（输入/输出） | 特点 |
|--------|------|------------------|------|
| 豆包 | Doubao-Pro | ¥0.8/¥2/百万tokens | 性价比高 |
| GLM | GLM-4-Flash | 免费 | 零成本 |
| DeepSeek | DeepSeek-Chat | $0.55/$2.19/百万tokens | 推理能力强 |
| 通义千问 | Qwen-Turbo | ¥0.3/¥0.6/百万tokens | 中文友好 |

## 三、非功能需求

### 3.1 性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 应用启动时间 | < 3秒 | 冷启动 |
| 笔记保存延迟 | < 500ms | 用户感知无延迟 |
| 本地AI响应 | < 3秒 | 首字响应 |
| 云端AI响应 | < 2秒 | 首字响应 |
| 语音转写 | < 3秒/分钟音频 | 实时转写 |
| 时间轴渲染 | < 1秒 | 100条记录内 |

### 3.2 可用性要求

| 指标 | 目标值 |
|------|--------|
| 系统可用性 | > 99.9%（本地模式） |
| 数据持久化 | 100%（自动保存） |
| 错误恢复 | 自动降级，不丢失数据 |

### 3.3 安全要求

- API Key使用AES-256加密存储
- 敏感数据不上传云端（本地模式）
- 支持数据本地备份
- 日志脱敏处理

### 3.4 兼容性要求

- **操作系统**：macOS 12.0+
- **硬件要求**：
  - 最低：8GB内存，4核CPU
  - 推荐：16GB内存，8核CPU，GPU（可选）

## 四、用户故事

### 4.1 核心用户故事

**US-01：快速记录投资想法**

```
作为 一名股票投资者
我想要 在看盘时快速记录投资想法
以便于 后续复盘分析

验收标准：
- 点击录音按钮，3秒内开始录音
- 录音结束后，自动转写为文字
- 转写结果自动填入编辑器
- 整个过程不影响看盘操作
```

**US-02：查看历史记录**

```
作为 一名股票投资者
我想要 查看某只股票的所有历史记录
以便于 回顾之前的判断

验收标准：
- 选择股票后，显示该股票的所有记录
- 按时间倒序排列
- 点击任意记录可查看详情
- 支持按时间范围筛选
```

**US-03：AI辅助整理**

```
作为 一名股票投资者
我想要 AI帮我整理口语化的录音内容
以便于 形成结构化的笔记

验收标准：
- 点击优化按钮，AI处理文本
- 去除"嗯"、"啊"等口语词
- 修正语法和标点
- 保持原文的投资逻辑
- 处理时间 < 3秒
```

**US-04：观点提取**

```
作为 一名股票投资者
我想要 AI自动提取我的投资观点
以便于 后续统计验证

验收标准：
- 自动识别看多/看空/中性
- 给出信心度评分（0-1）
- 识别时间维度（短线/中线/长线）
- 提取关键判断因素
```

**US-05：成本控制**

```
作为 一名成本敏感的用户
我想要 使用本地AI避免API费用
以便于 无限制使用

验收标准：
- 支持完全本地运行
- 无需网络连接
- 零API调用成本
- 性能满足基本需求
```

### 5.2 关键交互

**录音流程：**
1. 点击"录音"按钮 → 按钮变红，显示录音时长
2. 说话 → 实时显示音量波形
3. 点击"停止" → 保存音频，自动转写
4. 转写完成 → 文本填入编辑器

**AI优化流程：**
1. 输入/转写文本
2. 点击"优化"按钮
3. 显示处理中状态
4. 显示优化结果
5. 用户确认或修改
