# 面向对象笔记系统 - 产品需求文档（PRD）

**版本：** v2.0  
**状态：** 待评审  
**创建日期：** 2026-03-23  
**适用范围：** 下一阶段产品规划（在现有 v3.1 录音转写能力上演进）

---

## 一、产品定位

### 1.1 核心定位

这是一个“面向对象”的垂直话题笔记系统：

- 对象是核心组织单元（当前对象为 A 股股票）
- 一个对象对应一个笔记文件（文件即数据库）
- 每次新增信息都是一个“时间点事件”
- 每条事件允许用户配置观点（看多/看空/未知）
- 通过时间轴与复盘模块完成验证与统计

### 1.2 当前垂直话题

- 话题类型：A 股
- 对象主键：`stock_code`（如 `600519`）
- 对象别名：`stock_name`（如 `贵州茅台`）

### 1.3 未来扩展方向

架构需支持新增话题插件，例如：

- ETF
- 数字货币
- 宏观事件

扩展时保持“一个对象一个文件”的一致模型。

## 二、目标与价值

### 2.1 用户目标

- 快速记录交易想法，不打断观察流程
- 把口语化输入沉淀成可检索、可追踪的事件时间线
- 用事实（K 线）验证观点，计算胜率

### 2.2 产品价值

- **结构化沉淀**：时间点 + 观点 + 证据（原始文本/音频）
- **低心智负担**：对象即文件，定位简单
- **可复盘闭环**：记录 -> 对齐 -> 判定 -> 统计

## 三、核心需求（你定义的 6 项）

### 3.1 垂直话题与对象体系

1. 系统必须支持垂直话题配置。  
2. 初版内置 A 股话题，包含股票代码与股票名称映射。  
3. 所有录音和手动输入都必须绑定到一个对象（股票）。

### 3.2 一个对象一个笔记文件

1. 每只股票只有一个主笔记文件。  
2. 文件包含对象元信息与事件列表。  
3. 文件既是数据源，也是可读文档（Markdown + YAML Front Matter）。

### 3.3 时间点事件

1. 每次录音转文字或手动输入都创建一个事件。  
2. 默认事件时间为“创建时间”。  
3. 用户可以显式指定事件时间（绝对时间）。  
4. 需要保留两个时间字段：
- `event_time`（业务发生时间，可编辑）
- `created_at`（系统写入时间，不可编辑）
5. 时间输入粒度默认到分钟（`YYYY-MM-DD HH:mm`）。

### 3.4 观点配置（看多/看空/未知）

1. 每个事件支持观点字段：`bullish` / `bearish` / `unknown`。  
2. 默认值为 `unknown`。  
3. 用户可在创建时设置，也可在后续编辑修改。  
4. 可选附加字段：
- 预测周期（如 `T+1`, `3D`, `1W`）
- 判定规则模板（可继承全局默认）

### 3.5 时间轴展示（单对象）

1. 单对象时间轴模块，按时间展示全部事件。  
2. 时间轴需可视化区分观点。  
3. 支持快速筛选（时间范围、观点类型）。  
4. 时间轴节点支持跳转到对应事件详情并编辑观点。

### 3.6 独立复盘分析模块

1. 独立的复盘模块可加载单只股票 K 线。  
2. 将笔记事件时间与 K 线时间对齐。  
3. 用户可配置“如何判断正确”。  
4. 支持按时间区间统计观点胜率。
5. 支持按时间区间统计“所有股票”的综合胜率（简单汇总，不加权）。

## 七、复盘判定规则（需求定义）

### 7.1 默认规则（可配置）

- 看多命中：在指定窗口内，价格上涨幅度 >= 阈值
- 看空命中：在指定窗口内，价格下跌幅度 >= 阈值
- 未知：不计入胜率

### 7.2 用户可配置参数

- 判定窗口：默认 `3D`（短线预判）
- 价格基准：`收盘价` 或 `VWAP`
- 阈值：默认 `3%`
- K 线周期：支持切换，最小粒度 `5分钟`
- 是否排除停牌日
- 是否按交易日对齐

### 7.3 K 线数据源策略（已确认）

- 触发 API 拉取行情数据
- 首次拉取后本地持久化
- 后续优先读取本地缓存，按策略增量刷新
### 7.4 统计口径（从简）

- 样本数
- 命中数
- 胜率
- 分观点胜率（看多胜率、看空胜率）
- 全对象综合胜率（按时间区间，简单命中/样本）

### 7.5 默认口径（已确认）

- 默认判定窗口：`3D`
- 默认阈值：`3%`
- 默认支持 K 线周期切换，最小 `5分钟`
- `unknown` 默认不计入胜率
- 事件时间输入粒度：分钟级

## 八、非功能需求

### 8.1 性能

- 单对象 1,000 条事件，时间轴打开 < 1s
- 复盘页面切换对象，首屏图表 < 2s（本地缓存命中）

### 8.2 可用性

- 录音失败不能影响手动记录
- 任意编辑操作支持撤销/恢复（至少单步）

### 8.3 数据安全

- 本地优先存储
- 文件可读可迁移
- 提供对象级导出

## 十、待确认项（剩余）

无。

已确认的数据增长处理策略：

- 不采用按年分卷。
