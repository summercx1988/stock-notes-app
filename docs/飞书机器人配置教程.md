# 飞书机器人配置教程

## 零、重要说明

**在飞书开放平台中，创建"企业自建应用"就等于创建了一个机器人。**

- ✅ 不需要单独创建"机器人"
- ✅ 应用创建后自动拥有机器人能力
- ✅ 用户在飞书中搜索应用名称即可找到机器人

---

## 一、创建飞书企业自建应用（即创建机器人）

### 1.1 进入飞书开放平台

1. 打开浏览器，访问 [飞书开放平台](https://open.feishu.cn/)
2. 使用你的飞书账号登录
3. 点击右上角「开发者控制台」

### 1.2 创建应用

1. 点击「创建企业自建应用」
2. 填写应用信息：
   - **应用名称**：`股票笔记助手`（或你喜欢的名称）
   - **应用描述**：`远程录入股票投资笔记`
   - **应用图标**：上传一个图标（可选）
3. 点击「创建」

### 1.3 获取凭证

创建成功后，在「凭证与基础信息」页面可以看到：

| 凭证 | 说明 | 在应用中的位置 |
|------|------|----------------|
| **App ID** | 应用唯一标识 | 设置 → 飞书机器人 → App ID |
| **App Secret** | 应用密钥 | 设置 → 飞书机器人 → App Secret |

⚠️ **重要**：App Secret 只显示一次，请立即复制保存！

---

## 二、配置应用权限

### 2.1 添加权限

在左侧菜单点击「权限管理」→「申请权限」，搜索并添加以下权限：

| 权限名称 | 权限标识 | 用途 |
|----------|----------|------|
| 获取与发送消息 | `im:message` | 接收和发送消息 |
| 以应用身份发消息 | `im:message:send_as_bot` | 以机器人身份回复消息 |
| 获取用户基本信息 | `contact:user.base:readonly` | 获取用户信息（可选） |

### 2.2 发布版本

权限配置后需要发布版本才能生效：
1. 点击左侧「版本管理与发布」
2. 点击「创建版本」
3. 填写版本号（如 `1.0.0`）和更新说明
4. 点击「保存并发布」

---

## 三、配置事件订阅（WebSocket 模式）

### 3.1 开启事件订阅

1. 点击左侧「事件订阅」
2. 选择「使用长连接（WebSocket模式）」
   - ✅ **推荐**：无需公网 IP，适合本地开发
   - ❌ 不选「使用 Webhook 模式」

### 3.2 添加事件

点击「添加事件」，搜索并添加：

| 事件名称 | 事件标识 | 用途 |
|----------|----------|------|
| 接收消息 | `im.message.receive_v1` | 接收用户发送的消息 |
| 卡片交互回调 | `card.action.trigger` | 接收确认卡片、编辑卡片、候选股票卡片的按钮与表单提交回调 |

### 3.3 保存配置

点击「保存」按钮，系统会显示「连接配置」信息。

---

## 四、在 Electron 应用中配置

### 4.1 打开设置

1. 启动股票投资笔记应用
2. 点击右上角「工具」→「设置」
3. 切换到「飞书机器人」Tab

### 4.2 填写凭证

将飞书开放平台获取的凭证填入：

```
App ID:      cli_xxxxxxxxxxxx
App Secret:  xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 4.3 保存设置

点击「保存」按钮保存配置。

---

## 五、启用远程录入

### 5.1 开启服务

在应用 Header 右侧找到「远程录入」开关：
- 🔘 点击开关启用服务
- 🟢 绿色状态灯 = 已连接
- ⚪ 灰色状态灯 = 未连接

### 5.2 验证连接

如果开关无法开启，检查：
1. App ID 和 App Secret 是否正确
2. 应用是否已发布版本
3. 事件订阅是否已配置

---

## 六、使用机器人录入笔记

### 6.1 找到机器人

1. 在飞书中搜索你的机器人名称
2. 进入机器人对话框

### 6.2 发送笔记

直接发送文本消息，例如：

```
贵州茅台今天放量突破，看多，买入
```

### 6.3 确认保存

机器人会返回确认卡片：

```
📝 笔记确认
股票：贵州茅台 (600519)
观点：看多 | 操作：买入

[✓ 确认] [✏️ 修改] [❌ 取消]
```

点击「确认」即可保存笔记。

### 6.4 补充股票信息

如果 AI 无法识别股票，机器人会提示：

```
🔍 需要补充股票信息
未能识别股票信息，请提供股票名称或代码

请直接回复股票名称或代码，例如：贵州茅台 或 600519
```

直接回复股票名称或代码即可。

## 七、常见问题

### Q1: 开关无法开启，提示「未配置飞书应用」

**原因**：App ID 或 App Secret 未填写或填写错误

**解决**：
1. 检查设置中的凭证是否正确
2. 确认没有多余的空格
3. 重新从飞书开放平台复制凭证

### Q2: 开启后状态灯一直是灰色

**原因**：WebSocket 连接失败

**解决**：
1. 检查网络连接
2. 确认事件订阅已配置
3. 确认应用已发布版本
4. 查看控制台日志排查错误

### Q3: 发送消息没有响应

**原因**：事件未正确订阅或权限不足

**解决**：
1. 确认已添加 `im.message.receive_v1` 事件
2. 确认已添加 `card.action.trigger` 事件
3. 确认已添加 `im:message` 和 `im:message:send_as_bot` 权限
4. 确认应用版本已发布

### Q3.1: 编辑卡片点击保存时报错

**常见原因**：
1. 本地旧 Electron 进程没有退出，飞书仍连接到旧代码
2. 飞书开放平台没有订阅新版卡片交互事件 `card.action.trigger`

**建议排查**：
1. 完整关闭所有旧的 Electron 开发进程后重新启动
2. 在终端使用 `npm run electron:dev` 启动，观察主进程日志
3. 重新发送一条新消息，再点击“修改”和“保存”

### Q4: 股票识别不准确

**原因**：AI 提取依赖文本质量

**解决**：
1. 在消息中明确提及股票名称或代码
2. 使用完整的股票名称（如「贵州茅台」而非「茅台」）
3. 如果识别错误，可以通过对话补充正确信息

### Q5: 笔记保存失败

**原因**：本地文件写入问题

**解决**：
1. 检查数据目录是否有写入权限
2. 检查磁盘空间
3. 查看控制台日志排查错误

## 八、安全建议

### 8.1 密钥保护

- ⚠️ 不要将 App Secret 分享给他人
- ⚠️ 不要将凭证提交到代码仓库
- ⚠️ 定期在飞书开放平台重置 Secret

### 8.2 权限最小化

- 只申请必要的权限
- 定期检查权限列表
- 不需要的权限及时移除

### 8.3 访问控制

建议在飞书开放平台配置：
- 可用性设置 → 仅限企业内部员工使用
- IP 白名单（如有固定 IP）
