# 转写功能线说明

## 功能线概览

系统提供**两条独立的转写功能线**，满足不同使用场景：

```
┌─────────────────────────────────────────────────────────────────┐
│                        转写功能线                                │
└─────────────────────────────────────────────────────────────────┘

功能线一：流式转写（实时录音）
───────────────────────────────
录音 → 实时转写 → 显示文本 → AI分析 → 保存笔记
  ↓
每3秒转写一次
实时显示转写结果

功能线二：文件转写（上传音频）
───────────────────────────────
上传音频 → 完整转写 → 显示文本 → AI分析 → 保存笔记
  ↓
一次性转写完整文件
适合处理已有录音
```

## 功能线对比

| 特性        | 流式转写              | 文件转写              |
| --------- | ----------------- | ----------------- |
| **入口**    | "开始录音" 按钮        | "上传音频" 按钮        |
| **转写时机**  | 录音过程中实时转写        | 上传后一次性转写         |
| **显示方式**  | 逐段显示，实时更新        | 完整显示，一次性展示       |
| **适用场景**  | 实时记录投资想法         | 处理已有录音文件         |
| **延迟**    | ~3秒（每3秒转写一次）     | 取决于文件长度          |
| **UI状态**  | 显示"正在录音..."       | 显示"正在转写..."       |

## 流式转写流程

```
用户点击"开始录音"
    ↓
Swift服务启动AVAudioEngine
    ↓
音频数据收集到缓冲区
    ↓
每3秒触发一次转写
    ↓
创建临时WAV文件
    ↓
调用whisper.cpp转写
    ↓
发送transcript消息（isFinal=false）
    ↓
UI实时显示转写文本
    ↓
用户点击"停止录音"
    ↓
保存完整音频文件
    ↓
发送audio_saved消息
    ↓
自动进入AI分析流程
```

## 文件转写流程

```
用户点击"上传音频"
    ↓
选择本地音频文件
    ↓
发送transcribe_file消息
    ↓
Swift服务调用whisper.cpp
    ↓
一次性转写完整文件
    ↓
发送transcript消息（isFinal=true）
    ↓
UI显示完整转写文本
    ↓
进入AI分析流程
```

## 技术实现要点

### 流式转写

1. **音频收集**：使用AVAudioEngine实时收集音频数据
2. **定时转写**：每3秒触发一次转写任务
3. **临时文件**：创建临时WAV文件供whisper.cpp使用
4. **实时推送**：通过WebSocket实时推送转写结果
5. **状态管理**：使用`isFinal=false`标识中间结果

### 文件转写

1. **文件上传**：支持WAV/MP3/M4A/AAC格式
2. **一次性转写**：调用whisper.cpp转写完整文件
3. **结果推送**：使用`isFinal=true`标识最终结果

## 测试指南

### 流式转写测试

1. 点击"录音"按钮打开窗口
2. 选择"流式转写"模式
3. 点击"开始录音"
4. **对着麦克风说话**（必须有实际语音输入）
5. 观察实时转写结果显示
6. 点击"停止录音"
7. 验证音频文件保存
8. 验证自动进入AI分析

### 文件转写测试

1. 点击"录音"按钮打开窗口
2. 选择"文件转写"模式
3. 上传一个包含语音的音频文件
4. 点击"开始转写"
5. 验证转写结果显示
6. 验证进入AI分析流程

## 注意事项

1. **流式转写需要实际语音**：自动化测试无法提供语音输入，必须手动测试
2. **文件格式支持**：文件转写支持WAV/MP3/M4A/AAC格式
3. **模式切换**：用户可随时切换转写模式
4. **状态持久化**：录音过程中关闭窗口会自动停止录音
