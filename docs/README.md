## 一、快速开始

### 1.1 系统要求

- macOS 13.0 或更高版本
- Node.js 18 或更高版本
- ffmpeg（用于音频格式转换）

### 1.2 安装步骤

```bash
# 1. 克隆项目
git clone https://github.com/your-repo/stock-notes-app.git
cd stock-notes-app

# 2. 安装依赖
npm install

# 3. 安装 ffmpeg（如果未安装）
brew install ffmpeg

# 4. 编译 Swift 语音服务
cd ../voice-transcriber-service/Sources
swiftc -parse-as-library -o ../voice-transcriber-service *.swift \
  -framework AVFoundation -framework Foundation -framework Network \
  -framework CoreAudio -framework CoreFoundation

# 5. 下载 Whisper 模型
mkdir -p whisper.cpp/models
curl -L -o whisper.cpp/models/ggml-medium.bin \
  "https://hf-mirror.com/ggerganov/whisper.cpp/resolve/main/ggml-medium.bin"

# 6. 更新股票数据库
npm run update-stocks

# 7. 启动应用
GLM_API_KEY=your_api_key npm run electron:dev
```

## 二、功能介绍

### 2.1 语音录入

**启动录音**：
1. 点击界面右上角的 **录音** 按钮
2. 在弹窗中选择 **🎤 语音录入** 标签页
3. 点击 **开始录音** 按钮
4. 对着麦克风说话
5. 点击 **停止录音** 按钮

**实时转写**：
- 系统会在录音过程中实时显示转写文本
- 转写延迟约 1 秒

**AI 分析**：
- 自动识别股票名称和代码
- 自动提取投资观点（看多/看空/中性）
- 自动识别时间信息

### 2.2 手动输入

1. 点击 **录音** 按钮
2. 选择 **✏️ 手动输入** 标签页
3. 输入文本内容
4. 点击 **AI 分析** 按钮

### 2.3 上传音频文件

支持格式：
- WAV（推荐）
- MP3
- M4A（iPhone 录音格式）
- AAC

## 四、音频存储

### 4.1 存储位置

音频文件保存在：`~/Documents/VoiceTranscriber/audio/`

```
Documents/
└── VoiceTranscriber/
    └── audio/
        ├── recording_2025-03-20T09-30-00.wav
        ├── recording_2025-03-20T14-30-00.wav
        └── ...
```

### 4.2 自动清理

系统自动保留最近 100 个音频文件，超过限制时自动删除最旧的文件。

## 五、常见问题

### Q1: 录音没有声音？

检查系统偏好设置 → 安全性与隐私 → 隐私 → 麦克风，确保应用有麦克风权限。

### Q2: 转写结果不准确？

1. 确保使用 medium 或更大的模型
2. 说话时尽量清晰
3. 避免环境噪音

### Q3: Swift 服务启动失败？

检查端口 8765 是否被占用：
```bash
lsof -i :8765
```

如果被占用，可以修改配置文件 `~/.voice-transcriber/config.json` 中的端口。

### Q4: 如何更换 Whisper 模型？

1. 下载新模型到 `whisper.cpp/models/` 目录
2. 修改配置文件中的 `modelPath`

可用模型：
| 模型 | 大小 | 准确率 | 速度 |
|------|------|--------|------|
| tiny | 75MB | 一般 | 最快 |
| base | 142MB | 较好 | 快 |
| small | 466MB | 好 | 中等 |
| medium | 1.5GB | 很好 | 较慢 |
| large | 2.9GB | 最好 | 最慢 |
