# 股票操盘训练与风格建模系统 - 产品需求文档（PRD）

**版本：** v0.1  
**状态：** 需求梳理草案  
**创建日期：** 2026-03-27  
**适用范围：** `stock-notes-app` 后续独立项目 `future_work/trading-trainer`

---

## 一、产品定位

### 1.1 核心定位

这不是传统“模拟盘下单软件”，而是一个面向离散交易训练的 **盲训回放 + 风格建模系统**：

- 前台子系统负责“匿名 K 线逐根回放训练”
- 后台子系统负责“用户风格标注、扩样、训练集版本化与模型训练”
- 目标不是预测市场全部机会，而是沉淀并放大“用户自己能知行合一执行的风格”
- 训练完成后，系统从收益、时机、纪律、风格一致性四个维度复盘

### 1.2 解决的问题

现有模拟盘通常存在三个问题：

- 后续走势可见，容易“事后聪明”
- 股票代码、名称、日期暴露，容易带入先验记忆
- 复盘只看盈亏，缺少对买点、卖点和操作风格的归因

而单纯量化训练平台通常也存在三个问题：

- 不知道标签代表的是“理想化最优点”，还是“用户能执行的点”
- 缺少人机协同扩样机制，难以逐步贴近个人风格
- 缺少训练数据版本和模型版本的闭环管理

本产品要解决的是：**如何在接近真实不确定性的条件下，训练主观操盘能力，并沉淀为可迭代的个人风格模型。**

### 1.3 目标用户

- 主要用户：希望提升 A 股离散交易能力的个人交易者
- 次要用户：需要训练买卖节奏、纪律和复盘能力的研究型交易者

## 二、产品目标

### 2.1 用户目标

- 快速进入一段匿名市场样本并开始训练
- 逐根 K 线做买入、卖出、持有、跳过决策
- 用统一规则计算每次训练的收益与执行质量
- 长期沉淀自己的错误模式与优势风格
- 用“影子策略”对照自己的操作是否稳定
- 把自己认可且能执行的买卖点打标成训练数据
- 让系统模仿自己的风格去扩样，并通过抽样复核逐步提纯
- 基于不同版本数据集训练不同版本的买卖点预测模型

### 2.2 产品目标

- 建立严格的“未来信息隔离”
- 建立低延迟、强反馈的 bar-by-bar 操作体验
- 建立结构化复盘闭环：训练 -> 标注 -> 评分 -> 归因 -> 改进
- 建立结构化建模闭环：人工打标 -> 自动扩样 -> 人工抽检 -> 数据集版本 -> 模型训练 -> 回测评估
- 为后续扩展到 ETF、期货、数字货币保留模块边界

### 2.3 非目标

- 暂不做真实券商接入
- 暂不做 Level2 和逐笔撮合级别仿真
- 暂不做多人对战或社区排行榜
- 暂不把“自动量化交易”作为主产品主线

## 三、设计原则

- **先盲训，后解释**：先保护训练纯度，再做分析解释
- **先轻量闭环，后复杂仿真**：v1 优先 bar close 成交，不先做复杂撮合
- **先规则归因，后 AI 总结**：AI 负责语言归纳，不直接替代可解释规则
- **先单品种单会话，后组合训练**：v1 聚焦单只股票单次训练
- **先长期积累画像，后个性化推荐**：先收集足够行为数据，再做风格匹配

## 四、范围定义

### 4.1 V1 交易范围

- 市场：A 股现金股票
- 方向：仅做多，不做融券做空
- 会话：一次训练仅针对一只股票
- 仓位：默认单仓位模型，先支持“空仓 / 持仓”两态
- 成交：买入和卖出均按当前 bar 收盘价成交
- 手续费：支持配置，但默认不作为训练主评分权重

### 4.2 V1 时间粒度

- 支持 `1D / 60m / 30m / 15m / 5m`
- v1 默认推荐 `1D` 与 `15m`
- 最小粒度先不低于 `5m`

### 4.3 V1 会话结果

每次训练至少输出：

- 会话收益率
- 单笔交易收益率
- 最大有利波动（MFE）
- 最大不利波动（MAE）
- 持仓 bar 数
- 买点质量
- 卖点质量
- 过度交易/过早卖出/扛单等行为标签

### 4.4 V1 新增建模范围

- 先支持“买点 / 卖点 / 无操作”三类标签
- 标签语义优先定义为“我愿意执行且可复现执行的点”，而不是理论最优拐点
- 支持一份原始人工标签集，和基于该标签集扩展出的候选标签集
- 支持训练数据版本与模型版本的独立管理
- v1 先支持监督式离线训练，不做在线增量学习

## 五、核心业务需求

### 5.1 匿名训练与防泄漏

#### 必备要求

1. 隐藏股票代码、股票名称、行业、概念、是否自选等身份信息。
2. 默认隐藏绝对日期，改为 `Bar 1 / Bar 2 / Bar 3` 的相对时间轴。
3. 默认只展示当前 bar 及之前历史，不展示任何未来 K 线。
4. 训练过程中禁止通过 UI 直接搜索或揭示样本真实身份。

#### 建议新增要求

1. 提供两种匿名等级：
   - `严格匿名`：隐藏代码、名称、日期、绝对价格、成交额
   - `半匿名`：隐藏代码、名称，但保留日期与绝对价格
2. 严格匿名模式下可将首根可见 bar 价格基准化为 `100`，避免通过绝对价位猜股票。
3. 严格匿名模式下可将成交量显示为相对量能或分位数，而非绝对金额。
4. 默认从“非近期样本池”中抽样，避免用户对近半年热门走势有记忆。

### 5.2 样本抽样

#### 必备要求

1. 系统可随机选出一只股票并定位一个起点开始仿真。
2. 系统必须保证起点之前有足够历史 bar 供观察，起点之后有足够未来 bar 供评估。
3. 用户可选择“跳过该股票”并进入下一只样本。

#### 建议新增要求

1. 样本池支持过滤：
   - 非 ST
   - 非长期停牌
   - 上市天数大于阈值
   - 日均成交额高于阈值
2. 支持按市场状态采样：
   - 趋势上涨
   - 趋势下跌
   - 高波动震荡
   - 低波动盘整
3. 起始上下文默认展示最近 `100-300` 根历史 bar。
4. 每个样本都要记录“可训练窗口长度”和“机会强度”元数据，便于后续控制样本难度。

### 5.3 逐根回放与决策

#### 必备要求

1. 用户只能在当前 bar 收盘时做出决策。
2. 每推进一根 K 线，都要记录用户动作。
3. 当前会话支持以下动作：
   - `买入`
   - `卖出`
   - `继续持有`
   - `不操作`
   - `跳过该票`
4. 当用户持仓时，必须允许连续多个 bar 保持“持有”。
5. 会话可以因为以下原因结束：
   - 用户主动结束
   - 到达样本末尾
   - 用户跳过样本

#### 建议新增要求

1. 每个动作允许附带主观标签：
   - 入场理由
   - 出场理由
   - 风险级别
   - 信心等级
2. 每次买入前允许填写计划：
   - 预期方向
   - 预期持有周期
   - 预期止损/止盈逻辑
3. 提供两种推进模式：
   - `单步推进`：每次前进 1 根 bar
   - `快捷推进`：前进 N 根 bar，直到出现预警条件或用户动作

### 5.4 跳过样本的评估

#### 你的原始要求

- 如果一只股票没有交易，需要给出该区间的涨跌，作为决策成功率对比。

#### 需求补充

1. “跳过”不应直接计入交易胜率，否则会误伤保守但合理的风控行为。
2. 对跳过样本应单独输出：
   - 区间涨跌幅
   - 最大上涨幅度
   - 最大下跌幅度
   - 波动率
   - 是否存在明显趋势机会
3. 新增 `跳过质量` 指标，而不是简单记为“错过”：
   - `优质跳过`：波动混乱、机会不清晰、风险收益比差
   - `中性跳过`：存在机会但不明确
   - `错失机会`：后续出现高质量单边机会

### 5.5 评分与胜率

#### 必备要求

1. 会话结束后必须计算收益。
2. 胜率统计必须基于已发生交易样本，而不是所有抽样样本简单合并。
3. 每一笔交易都必须可追溯到入场、出场和当时所见 K 线状态。

#### 建议新增指标

- `交易胜率`：盈利笔数 / 已平仓笔数
- `会话收益率`
- `盈亏比`
- `平均持仓 bar 数`
- `买点效率`：从买入点到后续局部高点的捕获比例
- `卖点效率`：从局部高点到卖出点的偏离程度
- `持有纪律`
- `过度交易率`
- `错失趋势率`
- `跳过质量分`

### 5.6 复盘分析模块

#### 必备要求

1. 可以复盘用户的买点和卖点。
2. 可以分析成功和失败的共同点。
3. 可以查看某次训练中每个动作对应的 K 线位置。

#### 建议新增要求

1. 复盘分析分为三层：
   - `单会话复盘`
   - `多会话统计`
   - `风格画像`
2. 单会话复盘至少提供：
   - K 线回放轨迹
   - 每次动作的时点和标签
   - PnL 曲线
   - MFE/MAE 曲线
3. 多会话统计至少提供：
   - 高胜率入场模式
   - 高频失败模式
   - 平均持仓过长/过短
   - 常见止盈过早 / 止损过慢
4. 风格画像至少提供：
   - 趋势跟随偏好
   - 抄底摸顶偏好
   - 震荡内短打偏好
   - 加速段追涨倾向

### 5.7 影子策略对照

#### 必备要求

1. 除了用户分析外，系统还应给出可借鉴的量化思路。
2. 这些策略只作为影子策略，不自动替代用户决策。

#### 建议新增要求

1. 影子策略按风格分层，而不是一次塞入大量策略：
   - 趋势突破
   - 均线回踩续涨
   - 均值回归
   - 区间突破失败反转
2. 每个样本至少可选择 `1-3` 个影子策略做对照。
3. 对照维度包括：
   - 入场时机差异
   - 出场时机差异
   - 收益差异
   - 回撤差异
   - 交易次数差异
4. 长期训练后，系统可以给出：
   - “你更接近哪类策略风格”
   - “你常常在哪个环节偏离该风格”

### 5.8 macOS App 与性能

#### 必备要求

1. 产品优先做成 macOS 桌面 App。
2. K 线推进和动作反馈必须足够快，不能有明显卡顿。
3. 模块化设计，方便后续扩展策略、市场、数据源和更真实的撮合规则。

#### 性能目标

- 冷启动 `< 2s`
- 开始一场训练 `< 500ms`
- 单次“前进一步”反馈 `< 50ms`
- 单会话复盘加载 `< 300ms`
- 近 `1000` 场训练统计加载 `< 1s`

### 5.9 风格标注工作台

#### 必备要求

1. 用户可以在历史 K 线上手动打买点和卖点标签。
2. 标签必须允许附带上下文解释：
   - 为什么买
   - 为什么卖
   - 预期风格
   - 置信度
3. 标签必须支持版本管理，不能因为后续修正而覆盖历史版本。
4. 标签应区分来源：
   - 手工标签
   - 自动扩样标签
   - 抽检确认标签
   - 废弃标签

#### 评审补充

1. 标签语义必须先固定，否则后续模型会学歪。
2. 推荐把标签定义为：
   - `executable_buy`
   - `executable_sell`
   - `no_action`
3. 不建议直接标“理论最高点 / 最低点”，因为那通常不可执行，也不符合你的目标。

### 5.10 风格扩样与抽检

#### 必备要求

1. 系统需要一个方法，模仿用户风格去打标更多股票数据。
2. 自动扩样结果不能直接并入正式训练集，必须先进入候选池。
3. 用户必须可以抽样检验自动扩样质量。
4. 抽检结果必须回流到后续扩样和模型训练流程。

#### 建议新增要求

1. 扩样流程分成两阶段：
   - `候选发现`：找出疑似买卖点窗口
   - `候选打分`：给出更像你风格的置信度
2. 抽检采用主动学习思路，优先抽：
   - 模型最不确定的样本
   - 与历史风格冲突的样本
   - 高收益但低置信的样本
3. 自动扩样标签需要状态流转：
   - `proposed`
   - `accepted`
   - `rejected`
   - `needs_review`

### 5.11 训练数据集与版本管理

#### 必备要求

1. 支持多个训练数据版本并行存在。
2. 每个数据版本都必须记录：
   - 标签来源范围
   - 生成时间
   - 生成规则
   - 覆盖股票范围
   - 时间范围
   - 标签统计
3. 一个模型版本必须明确绑定一个或多个数据集版本。
4. 必须支持回溯“某个模型是由哪些数据训练出来的”。

#### 建议新增要求

1. 数据版本建议至少分三层：
   - `manual_seed`
   - `expanded_candidate`
   - `approved_training`
2. 数据版本应区分：
   - 全量样本数
   - 买点样本数
   - 卖点样本数
   - 无操作样本数
   - 被驳回样本数

### 5.12 模型训练与评估

#### 必备要求

1. 可以基于训练数据训练买点和卖点预测模型。
2. 支持多个模型版本并行存在。
3. 支持在未见过的时间区间和股票样本上做评估。
4. 模型输出必须可用于：
   - 给历史样本自动打候选标签
   - 在复盘时给出风格对照信号

#### 评审补充

1. v1 不建议直接追求“端到端深度学习预测最终收益”。
2. 更稳的路线是先做：
   - 样本窗口定义
   - 结构化特征
   - 二分类/多分类买卖点模型
   - 时间切分回测
3. 评估指标不能只看分类准确率，至少还要看：
   - Precision / Recall
   - 不同市场状态下表现
   - 生成信号后的交易收益
   - 与人工风格一致性
4. 模型必须支持废弃和回滚，不能只保留“最好”的一个。

## 七、核心流程

### 7.1 开始训练

```text
进入盲训工作台
  -> 选择训练模式 / 时间周期 / 难度
  -> 系统随机抽样
  -> 载入匿名K线历史窗口
  -> 用户开始逐根推进
```

### 7.2 训练过程

```text
显示当前bar
  -> 用户做决策
  -> 以当前bar收盘价成交
  -> 记录动作、仓位、标签
  -> 前进到下一bar
  -> 循环直到结束
```

### 7.3 训练结束

```text
结束会话
  -> 计算收益与行为指标
  -> 生成会话复盘
  -> 对照影子策略
  -> 累积到长期画像
```

### 7.4 风格建模流程

```text
用户在历史数据上手工打标
  -> 形成种子标签集
  -> 自动扩样生成候选标签
  -> 用户抽样复核
  -> 形成可训练数据集版本
  -> 训练模型版本
  -> 离线评估与时间切分回测
```

## 九、关键规则与边界

### 9.1 v1 明确规则

1. 只做多，不做空。
2. 只允许一只股票一个会话。
3. 买卖按当前 bar 收盘价成交。
4. 不做盘口级撮合。
5. 会话中途不揭示真实股票身份。
6. 跳过样本不进入交易胜率，但进入样本处理统计。
7. 打标和模型训练阶段也必须严格避免未来信息泄漏。
8. 一个标签必须对应固定的“可见历史窗口”，不能用未来数据回填特征。

### 9.2 需要预留但暂不做复杂化的规则

1. 涨跌停导致的无法成交
2. 停牌跨日处理
3. 除权除息处理
4. 多次加仓减仓
5. T+1 约束与更精细的交易费用
6. 多任务联合建模
7. 跨市场迁移学习
8. 在线学习与自动重训

## 十、优先级与迭代建议

### P0：先做出可训练闭环

1. 匿名样本抽样
2. 逐 bar 回放
3. 买入/卖出/持有/跳过
4. 收益统计
5. 单会话复盘
6. 手工风格打标
7. 标签语义与数据版本定义

### P1：把复盘做深

1. 入场/出场标签
2. MFE/MAE 分析
3. 跳过质量分析
4. 多会话聚合统计
5. 自动扩样候选池
6. 抽样审核流程

### P2：把训练做聪明

1. 用户风格画像
2. 影子策略对照
3. 相似风格策略推荐
4. 样本难度自适应
5. 离线模型训练
6. 数据集 / 模型版本管理

### P3：提高仿真真实性

1. A 股涨跌停成交约束
2. 停牌和除权处理
3. 分仓/加仓/减仓
4. 更细粒度撮合和交易成本模型
5. 更复杂的序列模型
6. 模型驱动的个性化样本推荐

## 十一、当前建议的产品结论

1. 这个项目现在应被定义为“训练系统 + 风格建模系统”的组合，而不是单一模拟盘。
2. v1 仍应聚焦“盲训 + 决策记录 + 复盘闭环 + 手工打标闭环”，不要一开始就做重型量化平台。
3. “跳过样本”必须单独建模，否则会把风控行为误算成能力差。
4. “风格标签”必须明确为“你愿意执行且可复现执行的点”，而不是理论最优点。
5. 自动扩样不能直接变成正式训练集，必须先经过候选池和抽样审核。
6. 数据集版本与模型版本必须从第一天就单独管理，否则后面很难解释模型退化和风格漂移。
7. 为了避免先验泄漏，匿名化不应只隐藏代码名称，最好同时支持隐藏日期和绝对价格。
